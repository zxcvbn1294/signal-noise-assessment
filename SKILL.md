# 資訊品質辨識框架 (Signal vs Noise Assessment)

## 概述

本框架用於系統化評估一篇文章、報導或觀點是「信號」（有價值的資訊）還是「噪音」（干擾判斷的雜訊）。

**核心原則**：噪音不是謊言，而是真假混合的資訊，用情緒包裝後難以分辨。框架的目標不是判斷「對錯」，而是評估「可信度」。

## 快速檢核清單（5秒版）

遇到任何需要評估的資訊時，先問這五個問題：

1. **這個人為什麼會知道這件事？**（來源可信度）
2. **他說錯了會付出什麼代價？**（利益衝突）
3. **什麼結果能證明他是錯的？**（可證偽性）
4. **有沒有獨立來源說同樣的事？**（交叉驗證）
5. **去掉情緒詞彙後還剩什麼？**（信息密度）

**決策規則**：如果有2個以上問題無法得到滿意答案，將該資訊標記為「待驗證」，不要基於它做重要決策。

## 完整評估框架：六層過濾體系

### 第一層：來源可信度評估 (Source Credibility)

#### 1.1 作者背景檢核表

| 檢查項目 | 信號特徵 | 噪音特徵 |
|---------|---------|---------|
| 專業經驗 | 在該領域有5年以上實務經驗 | 跨領域評論者，無直接經驗 |
| 利益位置 | 身處產業核心，有第一手資訊 | 純外部觀察者，依賴二手資訊 |
| 過往記錄 | 有可追溯的預測記錄，準確率可查 | 無歷史記錄或過往預測多次失準 |
| 聲譽代價 | 錯誤會損害其專業聲譽 | 錯誤無代價，可輕易改口 |

#### 1.2 關鍵問題

執行評估時，必須回答以下問題：

- 這個人為什麼會知道這件事？他的資訊來源是什麼？
- 他說錯了會付出什麼代價？是否有聲譽風險？
- 他有什麼動機可能讓他誇大或淡化事實？

#### 1.3 利益衝突檢測

**紅旗警示清單**：
- 作者正在銷售與文章主題相關的產品或服務
- 作者的公司估值/融資依賴於文章描述的趨勢成真
- 文章發布時機與某個商業事件（融資、產品發布、IPO）高度相關
- 作者有明確的意識形態立場可能影響客觀性

**計分標準**：
- 無明顯利益衝突：+2
- 有利益衝突但已披露：+1
- 有利益衝突未披露：-2
- 利益衝突是文章核心動機：-5

### 第二層：論證品質評估 (Argument Quality)

#### 2.1 邏輯結構檢核表

| 要素 | 信號特徵 | 噪音特徵 |
|-----|---------|---------|
| 因果關係 | 明確解釋A如何導致B | 僅呈現相關性，暗示因果 |
| 反面論證 | 主動討論可能的反駁 | 只呈現支持自己觀點的證據 |
| 不確定性 | 明確標示哪些是推測 | 將推測表述為確定事實 |
| 範圍限定 | 說明結論適用的條件和邊界 | 過度概化，聲稱普遍適用 |

#### 2.2 常見邏輯謬誤檢測清單

在評估文章時，主動掃描以下謬誤模式：

**訴諸權威謬誤**：
- 模式："X公司的CEO說..."、"諾貝爾獎得主認為..."
- 問題：權威人士可能有動機誇大，或在非專業領域發言
- 檢測方法：問「這個人在這個具體問題上是否真的是專家？」

**滑坡謬誤**：
- 模式："如果A發生，必然導致B，然後C，最終D"
- 問題：每一步的概率都未論證，連乘後概率可能很低
- 檢測方法：要求每一步的概率估計和論證

**倖存者偏差**：
- 模式："這些成功公司都用了這個方法"
- 問題：忽略了用同樣方法失敗的案例
- 檢測方法：問「那些失敗的案例呢？」

**虛假二分法**：
- 模式："要麼擁抱AI，要麼被淘汰"
- 問題：忽略中間的多種可能性
- 檢測方法：列出被忽略的中間選項

**訴諸情緒**：
- 模式：大量使用"震撼"、"恐怖"、"不可思議"等詞彙
- 問題：用情緒替代論證
- 檢測方法：去掉情緒詞彙後看還剩什麼實質內容

---

### 第三層：證據品質評估 (Evidence Quality)

#### 3.1 數據來源層級分類

| 層級 | 類型 | 可信度 | 使用建議 |
|-----|------|-------|---------|
| A級 | 官方技術文檔、經同行評審的論文、監管機構報告 | 高 | 可作為核心論據 |
| B級 | 有聲譽的研究機構報告、財報數據、專利文件 | 中高 | 可作為支持論據 |
| C級 | 新聞報導、業內人士訪談 | 中 | 需多源交叉驗證 |
| D級 | 社交媒體發言、匿名來源、「據悉」類表述 | 低 | 僅供參考，不可作為決策依據 |
| E級 | 無來源的斷言、「眾所周知」類表述 | 極低 | 基本忽略 |

#### 3.2 數據品質檢核問題

對於文章中的每一個關鍵數據點，問：

1. 這個數據的**原始來源**是什麼？（不是引用的媒體，而是數據的產生者）
2. 數據的**測量方法**是什麼？定義是否清晰？
3. **樣本量**多大？是否有選擇偏差？
4. 數據的**時間範圍**是什麼？是否還有時效性？
5. 是否有**其他獨立來源**可以交叉驗證？

#### 3.3 證據與結論匹配度紅旗

- 用少量個案推導普遍結論
- 證據只覆蓋部分論點，其他部分靠推測填充
- 關鍵論點缺乏直接證據，依賴「合理推斷」
- 數據呈現方式有選擇性（只展示支持結論的部分）

---

### 第四層：預測品質評估 (Prediction Quality)

#### 4.1 預測具體性檢核表

| 維度 | 信號特徵 | 噪音特徵 |
|-----|---------|---------|
| 時間框架 | "2026年Q2之前" | "即將"、"不久的將來" |
| 可驗證性 | 明確說明什麼結果能證明預測錯誤 | 模糊到無法被證偽 |
| 概率表述 | "我認為有60-70%的可能性" | "一定會"、"必然" |
| 條件說明 | "如果X和Y條件成立，則..." | 無條件的絕對預測 |

#### 4.2 可證偽性測試

好的預測必須能回答以下問題：

- 什麼樣的結果會證明這個預測是**錯**的？
- 在什麼**時間點**我們能判斷這個預測的對錯？
- 預測者願意在預測錯誤時**承認**嗎？

**警示信號**：
- "長期來看一定會發生"（無時間限制 = 無法被證偽）
- "可能會很快，也可能需要更長時間"（無論何時發生都算對）
- 預測失準後解釋為"時間推遲了但方向是對的"

---

### 第五層：情緒與理性比例 (Emotion vs Rationality)

#### 5.1 情緒詞彙密度檢測

**高情緒詞彙分類**：

| 類別 | 詞彙示例 |
|-----|---------|
| 恐懼類 | 崩塌、災難、末日、危機、恐怖、毀滅 |
| 興奮類 | 革命、顛覆、震撼、不可思議、史無前例、改變一切 |
| 緊迫類 | 立刻、馬上、不能錯過、最後機會、現在就要 |

**評估方法**：統計文章中情緒詞彙的出現頻率。如果每100字出現超過3個高情緒詞彙，需要提高警惕。

#### 5.2 敘事結構分析

以下敘事技巧能提升傳播力，但往往犧牲準確性：

- **過度依賴類比**："就像2020年2月的疫情前夕"
- **英雄/反派框架**："覺醒者" vs "沉睡的大眾"
- **末日/烏托邦敘事**：極端化的未來描繪
- **優越感敘事**："你比別人知道得早"、"看懂的人不到1%"

---

### 第六層：交叉驗證 (Cross-Validation)

#### 6.1 多源驗證矩陣

對於關鍵論點，填寫以下矩陣：

| 驗證來源 | 是否支持 | 來源獨立性 | 備註 |
|---------|---------|-----------|-----|
| 原始技術文檔 | | | |
| 學術論文 | | | |
| 競爭對手觀點 | | | |
| 持反對意見的專家 | | | |
| 歷史類比的實際結果 | | | |

**關鍵**：來源必須真正獨立。如果所有支持論點的來源最終都追溯到同一個原始來源，那只算一個來源。

#### 6.2 反向搜索策略

主動尋找反面觀點：

- 搜索 "[主題] criticism" 或 "[主題] skeptic"
- 尋找該領域有聲譽但持不同意見的專家
- 檢查預測者過去的預測記錄
- 搜索 "[預測者名字] wrong" 或 "[預測者名字] failed prediction"

## 實用評分卡

對每篇文章進行以下評分（每項-2到+2分）：

| 維度 | 評分標準 | 分數 |
|-----|---------|-----|
| **來源可信度** | 作者是否有直接經驗和聲譽代價 | |
| **利益衝突** | 是否有未披露的利益衝突 | |
| **邏輯結構** | 論證是否嚴謹，是否考慮反面 | |
| **證據品質** | 數據來源是否可靠可追溯 | |
| **預測具體性** | 預測是否可驗證可證偽 | |
| **情緒比例** | 理性分析vs情緒渲染的比例 | |
| **交叉驗證** | 是否有獨立來源支持關鍵論點 | |

**總分解讀**：
- **10-14分**：高品質信號，值得認真對待並納入決策考量
- **5-9分**：中等品質，需要進一步驗證後再做決策
- **0-4分**：噪音成分較多，謹慎參考，不建議作為決策依據
- **負分**：高概率噪音，建議忽略

## 特定領域的額外考量

### 科技趨勢類文章

額外檢查清單：
- [ ] 技術細節是否準確？（可找技術背景的人驗證）
- [ ] 是否混淆了「技術上可行」和「經濟上可行」？
- [ ] 是否考慮了採納曲線的摩擦（監管、組織慣性、社會信任）？
- [ ] 歷史上類似的技術預測，準確率如何？
- [ ] 是否有具體的成本/效益分析？

### 投資/金融類文章

額外檢查清單：
- [ ] 作者是否持有相關部位？
- [ ] 是否在特定時間點發布（財報前後、融資前後）？
- [ ] 是否只呈現有利數據，忽略不利數據？
- [ ] 風險提示是否充分？
- [ ] 是否有回測數據或歷史驗證？

### 產業預測類文章

額外檢查清單：
- [ ] 預測者過去在該產業的預測記錄如何？
- [ ] 是否考慮了黑天鵝事件的可能性？
- [ ] 是否過度線性外推當前趨勢？
- [ ] 是否考慮了競爭對手和替代方案？

## 元認知：檢查自己的偏見

**在評估他人文章之前，先問自己**：

1. **確認偏誤**：我是否因為這篇文章符合我的既有觀點，而降低了評估標準？
2. **權威崇拜**：我是否因為作者的名氣或地位，而跳過了批判性審視？
3. **新奇偏好**：我是否因為論點新穎刺激，而忽略了證據不足的問題？
4. **恐懼驅動**：我是否因為害怕錯過（FOMO），而急於接受未經驗證的結論？
5. **社會證明**：我是否因為很多人轉發分享，而假設內容可靠？

**校準方法**：問自己「如果這篇文章的結論與我的直覺相反，我還會用同樣的標準評估它嗎？」

## 使用注意事項

1. **框架的局限性**：本框架偏向保守，可能會過濾掉一些真正重要但表達方式不夠嚴謹的早期信號。

2. **非二元判斷**：信號和噪音不是二元的，大多數資訊是混合的。關鍵是提取有價值的部分，而非簡單地「接受」或「拒絕」。

3. **持續校準**：根據你自己的使用經驗，調整各項權重。記錄你的評估和後續結果，以改進判斷能力。

4. **輔助而非替代**：這個框架是輔助工具，不是替代人類判斷的算法。最終決策仍需要你的智慧。
