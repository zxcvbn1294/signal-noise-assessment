# 資訊品質辨識 - 評估範例

本文檔提供詳細的評估範例，展示如何應用框架分析真實文章。

---

## 範例一：AI趨勢預測文章

### 文章概述
- **標題**：Something Big Is Happening
- **作者**：Matt Shumer（HyperWrite CEO）
- **核心論點**：AI發展速度遠超大眾認知，遞歸式自我提升循環可能在12個月內啟動
- **傳播數據**：24小時內獲得7000萬閱讀

### 逐層評估

#### 第一層：來源可信度 (+1)

**正面因素**：
- 作者是AI公司CEO，有6年行業經驗
- 文章引用了Jimmy Ba（Adam優化器共同發明者）的離職警告
- 引用了Anthropic CEO Dario Amodei的類似觀點

**負面因素**：
- 作者不是核心技術研發人員，而是應用層CEO
- 主要依賴二手資訊和公開文檔

**評分理由**：有行業經驗但非技術核心，引用的專家來源較強，總體可信度中等偏上。

#### 第二層：利益衝突 (0)

**分析**：
- HyperWrite是AI公司，公司估值直接受益於AI熱潮
- 文章沒有直接推銷產品
- 作者聲稱「不想再對身邊人撒謊」，但這種敘事本身可能是行銷策略

**評分理由**：有明確利益衝突但未直接商業化，中性評分。

#### 第三層：論證品質 (+1)

**正面因素**：
- 提出了具體的時間線（2022-2026年AI能力演進）
- 文章後半部分討論了算力成本約束（平衡觀點）
- 引用了METR機構的具體測量數據

**負面因素**：
- 「遞歸自我提升→智能爆炸」的推論有滑坡謬誤傾向
- 「AI開始自我構建」的表述過度簡化了實際情況
- 使用了虛假二分法：「要麼擁抱AI，要麼被淘汰」

**評分理由**：有實質論證但存在明顯邏輯問題。

#### 第四層：證據品質 (+1)

**證據清單**：
| 論點 | 證據來源 | 層級 |
|-----|---------|-----|
| GPT-5.3-Codex參與自身開發 | OpenAI官方文檔 | A級 |
| AI獨立工作時間增長趨勢 | METR機構測量 | B級 |
| Anthropic代碼由AI編寫 | CEO公開發言 | C級 |
| 2020年2月類比 | 作者主觀判斷 | E級 |

**評分理由**：核心論點有A/B級證據支持，但部分推論依賴低級證據。

#### 第五層：預測品質 (+1)

**具體性分析**：
- 「12個月內上線」— 有時間框架，可驗證
- 「2026年將是決定性一年」— 有時間框架
- 「可能」、「或許」— 有不確定性表述

**問題**：
- 「智能爆炸」的定義模糊，難以判斷是否發生
- 缺乏明確的「什麼情況算預測失敗」的說明

**評分理由**：有時間框架但驗證標準不夠清晰。

#### 第六層：情緒比例 (-1)

**情緒詞彙統計**（摘錄）：
- 「震撼」出現2次
- 「恐怖」出現1次
- 「崩塌」出現1次
- 「不可思議」出現1次
- 疫情類比 — 強情緒敘事技巧

**敘事結構問題**：
- 「不想再對身邊人撒謊」— 建立內部人優越感
- 「看得懂看不懂無所謂，重要的是比身邊人看的快」— FOMO操作
- 英雄/覺醒者敘事框架

**評分理由**：情緒渲染成分較高。

#### 第七層：交叉驗證 (+1)

**驗證矩陣**：
| 來源 | 支持？ | 獨立性 |
|-----|-------|-------|
| OpenAI文檔 | 是 | 高 |
| Anthropic CEO | 是 | 高 |
| METR數據 | 是 | 高 |
| Jimmy Ba | 是 | 高 |
| 持反對意見專家 | 未引用 | - |

**問題**：文章只引用了同一立場的來源，缺乏反面觀點。

**評分理由**：多個獨立來源但立場單一。

### 總分計算

| 維度 | 分數 |
|-----|-----|
| 來源可信度 | +1 |
| 利益衝突 | 0 |
| 論證品質 | +1 |
| 證據品質 | +1 |
| 預測具體性 | +1 |
| 情緒比例 | -1 |
| 交叉驗證 | +1 |
| **總分** | **4** |

### 結論

**評級**：中等偏下（0-4分區間）

**處理建議**：
1. **提取**：OpenAI文檔、METR數據、專家引述是有價值的信號
2. **忽略**：情緒敘事、疫情類比、FOMO語言
3. **待驗證**：「遞歸自我提升」的時間表和「智能爆炸」的必然性
4. **行動**：不建議基於此文章做重大決策，但值得追蹤文中提到的數據來源

## 範例二：投資類文章（對比案例）

### 高品質範例特徵

一篇高品質的投資分析文章應該：

**來源可信度**：
- 作者披露自己是否持有相關部位
- 有可追溯的過往預測記錄
- 明確說明資訊來源

**論證品質**：
- 主動討論風險和反面觀點
- 區分事實和推測
- 提供多種情境分析

**證據品質**：
- 引用原始財報數據而非媒體摘要
- 提供數據來源連結
- 討論數據的局限性

**預測品質**：
- 給出具體的價格區間和時間框架
- 說明什麼情況會讓預測失效
- 提供概率估計而非絕對結論

**情緒比例**：
- 避免「必漲」「暴跌」等極端用詞
- 保持分析性而非鼓動性語調
- 不使用FOMO語言

### 低品質範例紅旗

以下特徵通常表示文章品質較低：

- 「內部消息」、「不為人知的秘密」
- 「現在不買就來不及了」
- 只呈現有利數據，忽略風險
- 無法追溯的數據來源
- 作者背景模糊或無法驗證
- 與特定商業事件時間點高度重合

## 範例三：科技預測歷史對照

### 過往預測準確度案例

用歷史案例校準你的判斷：

**自動駕駛**：
- 2015年預測：「2020年實現完全自動駕駛」
- 實際情況：2025年仍處於L2-L3級別
- 教訓：技術演示≠大規模部署

**放射科AI**：
- 2016年預測：「放射科醫生將在5年內被取代」
- 實際情況：放射科醫生就業人數反而增長
- 教訓：技術能力≠經濟替代

**量子計算**：
- 2010年代預測：「量子霸權將很快實現」
- 實際情況：量子霸權實現但實用化仍遠
- 教訓：里程碑≠商業化

**啟示**：當評估AI預測時，記住過往類似預測的準確率，並對時間表持謹慎態度。
